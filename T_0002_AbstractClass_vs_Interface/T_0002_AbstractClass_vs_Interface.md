# Тема 2. Абстрактные классы и интерфейсы

Абстрактный класс Vehicle — это "шаблонный" родитель. Он описывает то, что является общей сущностью для всех транспортов.

    Он может иметь поля (int Weight), реализованные методы (public void StartEngine() { ... }) и абстрактные методы (public abstract void Move();), которые должны быть реализованы потомками.

    Ключевая мысль: Абстрактный класс определяет отношение "is-a" (является). Car является Vehicle. Он выражает родственную связь по вертикали.

Интерфейс IFlyable — это контракт на способность. Он не говорит, что ты такое, он говорит, что ты умеешь делать.

    Он содержит только сигнатуры методов и свойств (void Fly();). Реализация — целиком на том, кто этот интерфейс подписал.

    Ключевая мысль: Интерфейс определяет отношение "can-do" (может делать). Airplane может IFlyable. Helicopter может IFlyable. И даже Bird (которая не является Vehicle) может IFlyable. Это горизонтальная связь.
